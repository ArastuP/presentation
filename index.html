Meet Arastu

Arastu is a particle physicist working on high-energy collision experiments.

His day job: hunt for new particles hiding in billions of collision events.

His tools: detectors the size of cathedrals, supercomputers, and a very good terminal.

The Scale of the Problem

  • The LHC produces ~1 petabyte of data per second at collision
  • After filtering: ~15 petabytes stored per year
  • Arastu's analysis alone touches millions of events
  • You cannot manage this with a mouse and folders named final_v3_REAL_final.csv

The Unix Shell — Arastu's Best Friend

A command-line interpreter that lets you talk directly to the operating system.

CERN's computing clusters (lxplus) run Linux. There is no GUI.

To run his simulation, Arastu types:
./run_simulation.sh --events 1000000 --energy 13TeV

Without the shell, Arastu cannot submit jobs, move data, or run anything at scale.

Physics Software Lives on the Command Line

  • ROOT — CERN's data analysis framework
  • Geant4 — particle detector simulations
  • MadGraph — event generators for new physics models
  • HTCondor / Slurm — cluster job schedulers
  • All of them: terminal only.

What is Version Control?

A system that records changes to files over time so you can recall specific versions later.

Think of it as an unlimited, labelled undo history — for your entire project.

Git is the most widely used version control system in the world.

Why Arastu Needs Version Control

Last Tuesday, Arastu's energy cut was pT > 25 GeV.
Today it's pT > 30 GeV.
His plot changed. Which result does he report?

Without version control: chaos, commented-out code, files like analysis_v7_corrected_FINAL2.py

With Git: every change is logged, labelled, and reversible.

Is Git an Important Skill?

ATLAS and CMS — the two largest LHC experiments — both manage their software on Git and GitLab.

Arastu's analysis framework has thousands of contributors across dozens of countries.

Without Git, coordinating that collaboration would be impossible.

Yes. It is essential.

Tidy Data in Particle Physics

  • Each row = one collision event
  • Each column = one variable (pT, eta, phi, mass, label)
  • Consistent types — no mixing strings and floats in the same column
  • Tidy data makes Arastu's Python analysis reproducible and shareable

Python — Arastu's Analysis Language

Python + uproot reads ROOT files directly — no C++ required.

awkward-array handles jagged event data elegantly.

matplotlib / mplhep produces publication-quality physics plots.

VSCode gives Arastu linting, Git integration, and Jupyter notebooks — all in one place.
